# 前后端接口对齐问题汇总报告

## 📋 概述

本报告基于对前端（React/TypeScript）和后端（Python/FastAPI）代码的全面扫描，识别出前后端接口对齐的所有问题。项目存在**严重的前后端接口不匹配**，导致无法正常联调运行。

---

## 🔴 严重问题（阻塞性）

### 1. 后端启动失败：向量维度超限

**问题描述**：后端服务无法启动，报错 `column cannot have more than 2000 dimensions for hnsw index`

**根本原因**：
- `.env` 配置 `EMBEDDING_DIM=2048`
- 数据库schema定义 `VECTOR(2048)` (database.py 第89、113、149行)
- Neon PostgreSQL 的 pgvector 扩展限制 HNSW 索引最多 **2000维**

**影响**：后端完全无法启动，所有API不可用

**修复方案**：
1. 修改 `.env` 文件：`EMBEDDING_DIM=1536`（或更小）
2. 修改 `api/core/database.py` 中所有 `VECTOR(2048)` 为 `VECTOR(1536)`
3. 清除已存在的数据库表并重新初始化

---

### 2. API响应包装层不匹配

**问题描述**：前端期望统一的响应包装格式，后端直接返回业务数据

**前端期望** (client.ts 第77-78行)：
```typescript
interface ApiResponse<T> {
  success: boolean;
  data: T;
  error: ApiError | null;
  timestamp: string;
}
// 前端取值：response.data
```

**后端实际**：
```python
# 直接返回业务数据，无包装层
return {"id": "xxx", "filename": "test.pdf", ...}
```

**影响**：所有API调用都会返回 `undefined`，前端无法获取数据

**修复方案**（二选一）：
- **方案A**：后端统一包装响应（推荐）
  ```python
  # api/core/response.py
  def success_response(data):
      return {"success": True, "data": data, "error": None, "timestamp": datetime.now().isoformat()}
  ```
- **方案B**：前端去掉 `.data` 取值
  ```typescript
  // client.ts 第78行改为
  return await response.json() as T;  // 直接返回，不取 .data
  ```

### 3. API基础路径重复

**问题描述**：前端BASE_URL包含 `/api`，后端路由也包含 `/api` 前缀

**前端配置** (client.ts 第3行)：
```typescript
const BASE_URL = import.meta.env.VITE_API_URL ?? 'http://localhost:8000/api';
```

**后端路由** (main.py 第65-68行)：
```python
app.include_router(textbooks.router, prefix="/api/textbooks", tags=["教材管理"])
```

**实际请求路径**：`http://localhost:8000/api/api/textbooks` ❌

**修复方案**：
- 修改前端 `BASE_URL` 为 `http://localhost:8000`
- 或修改后端路由前缀去掉 `/api`
