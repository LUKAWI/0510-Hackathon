## 部署方案选择

在魔搭社区部署时，你有两种方案可选：

### 方案一：前后端一起部署（推荐）

**优点：** 简单，只需一个创空间，无需处理跨域问题
**缺点：** 资源占用较高，需要同时运行前端和后端

**适用场景：** 快速演示、小型项目

### 方案二：前后端分离部署

**优点：** 资源隔离，可独立扩展
**缺点：** 需要处理跨域，配置复杂

**适用场景：** 生产环境、大型项目

本教程以 **方案一** 为主进行说明。

## 项目概述

**学科知识整合智能体** 是一个基于 AI 的教育工具，主要功能包括：

- 📚 多格式教材加载（PDF、Markdown、DOCX）
- 🔗 知识图谱自动构建与可视化
- 🤖 RAG 智能问答
- 💬 多轮对话支持
- 📊 跨教材知识整合

**技术栈：**
- 前端：React 18 + TypeScript + Vite + Tailwind CSS + AntV G6
- 后端：Python FastAPI + 通义千问 API + LangChain
- 数据库：Neon PostgreSQL + pgvector

## 第一步：准备数据库

### 1.1 注册 Neon 数据库

1. 访问 https://neon.tech 并注册账号
2. 创建新项目（Project）
3. 选择地区建议：**Singapore (ap-southeast-1)**（亚太地区，延迟较低）
4. 记录数据库连接串（Connection String）

### 1.2 获取数据库连接串

在 Neon 控制台中：

1. 进入你的项目
2. 点击 **Dashboard** → **Connection Details**
3. 复制连接串，格式如下：

```
postgresql://username:password@hostname/database?sslmode=require
```

**示例：**
```
postgresql://neondb_owner:npg_xxxxx@ep-xxx-pooler.c-2.ap-southeast-1.aws.neon.tech/neondb?sslmode=require
```

### 1.3 启用 pgvector 扩展

在 Neon SQL Editor 中执行：

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

验证扩展已启用：

```sql
SELECT * FROM pg_extension WHERE extname = 'vector';
```

## 第三步：项目配置

### 3.1 克隆项目

```bash
# 克隆项目到本地
git clone https://github.com/LUKAWI/0510-Hackathon.git
cd 0510-Hackathon
```

### 3.2 安装依赖

```bash
# 安装前端依赖
npm install

# 安装后端依赖（可选，本地测试用）
pip install -r api/requirements.txt
```

### 3.3 配置环境变量

复制环境变量模板：

```bash
cp .env.example .env
```

编辑 `.env` 文件，填入你的配置：

```env
# 通义千问 API Key（必填）
DASHSCOPE_API_KEY=sk-你的API Key

# Neon 数据库连接串（必填）
DATABASE_URL=postgresql://username:password@hostname/database?sslmode=require

# Embedding 模型配置
EMBEDDING_MODEL_NAME=text-embedding-v4
EMBEDDING_DIM=1024

# RAG 分块参数
CHUNK_SIZE=500
CHUNK_OVERLAP=80

# LLM 模型
LLM_MODEL_NAME=qwen3.5-plus

# 前端 API 地址（魔搭部署时留空，使用相对路径）
VITE_API_URL=
```

**重要说明：** 在魔搭部署时，`VITE_API_URL` 应该留空或设置为相对路径 `/api/`，因为前后端部署在同一个域名下。

### 3.4 配置后端静态文件服务（方案一必需）

如果选择前后端一起部署，需要修改 `api/main.py` 添加静态文件服务：

```python
# 在文件末尾添加以下代码（@app.get("/") 之前）

# 静态文件服务（魔搭部署时使用）
DIST_DIR = Path(__file__).parent.parent / "dist"
if DIST_DIR.exists():
    app.mount("/assets", StaticFiles(directory=DIST_DIR / "assets"), name="assets")
    
    @app.get("/{full_path:path}")
    async def serve_spa(full_path: str):
        """服务前端 SPA 页面"""
        file_path = DIST_DIR / full_path
        if file_path.is_file():
            return FileResponse(file_path)
        return FileResponse(DIST_DIR / "index.html")
```

### 3.5 更新魔搭部署配置

编辑 `ms_deploy.json` 文件，更新环境变量：

```json
{
  "$schema": "https://modelscope.cn/api/v1/studios/deploy_schema.json",
  "sdk_type": "docker",
  "resource_configuration": "platform/2v-cpu-16g-mem",
  "port": 7860,
  "environment_variables": [
    { "name": "DASHSCOPE_API_KEY", "value": "sk-你的API Key" },
    { "name": "DATABASE_URL", "value": "你的数据库连接串" },
    { "name": "EMBEDDING_MODEL_NAME", "value": "text-embedding-v4" },
    { "name": "EMBEDDING_DIM", "value": "1024" },
    { "name": "CHUNK_SIZE", "value": "500" },
    { "name": "CHUNK_OVERLAP", "value": "80" },
    { "name": "LLM_MODEL_NAME", "value": "qwen3.5-plus" }
  ]
}
```

### 3.5 更新 CORS 配置

编辑 `api/main.py`，添加魔搭社区域名到 CORS 白名单：

```python
# CORS 中间件（允许前端域名 + 本地开发 + 魔搭社区）
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:5173",
        "https://modelscope.cn",           # 魔搭社区
        "https://*.modelscope.cn",          # 魔搭子域名
        "https://modelscope.aliyun.com",    # 魔搭阿里云版
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 3.6 本地构建测试

```bash
# 构建前端
npm run build

# 验证构建产物
ls -la dist/
```

### 3.7 更新 Dockerfile（方案一必需）

如果选择前后端一起部署，需要更新 `backend.Dockerfile`：

```dockerfile
FROM modelscope-registry.cn-beijing.cr.aliyuncs.com/modelscope-repo/python:3.10

WORKDIR /home/user/app

# 安装 Node.js
RUN curl -fsSL https://deb.nodesource.com/setup_18.x | bash - && \
    apt-get install -y nodejs && \
    apt-get clean && \
    rm -rf /var/lib/apt/lists/*

# 复制前端文件
COPY package.json package-lock.json ./
COPY src/ ./src/
COPY public/ ./public/
COPY index.html vite.config.ts tsconfig.json tsconfig.app.json tsconfig.node.json components.json ./

# 安装前端依赖并构建
RUN npm install --registry=https://registry.npmmirror.com && \
    npm run build

# 安装后端依赖
COPY api/requirements.txt ./api/
RUN pip install --no-cache-dir -r api/requirements.txt -i https://mirrors.aliyun.com/pypi/simple/

# 复制后端源码
COPY api/ ./api/

# 复制配置文件
COPY ms_deploy.json ./

# 暴露端口
EXPOSE 7860

# 设置环境变量
ENV PYTHONPATH=/home/user/app

# 启动命令
CMD ["uvicorn", "api.main:app", "--host", "0.0.0.0", "--port", "7860"]
```

## 第四步：创建魔搭创空间

### 4.1 登录魔搭社区

1. 访问 https://www.modelscope.cn
2. 使用阿里云账号登录

### 4.2 创建创空间

1. 点击右上角头像 → **我的创空间**
2. 点击 **创建新的创空间**
3. 填写配置：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 创空间名称 | knowledge-agent | 自定义名称 |
| 描述 | 学科知识整合智能体 | 简要描述 |
| SDK 类型 | **Docker** | 必须选择 Docker |
| 资源配置 | **2vCPU, 16GB 内存** | 推荐配置 |
| 可见性 | 公开/私有 | 按需选择 |

**资源配置建议：**
- **最低配置：** 2vCPU, 8GB 内存（仅后端 API）
- **推荐配置：** 2vCPU, 16GB 内存（前后端一起）
- **高性能：** 4vCPU, 32GB 内存（大量并发）

4. 点击 **创建**

### 4.3 配置创空间环境变量

在创空间设置页面，找到 **环境变量** 配置区域，添加以下变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| DASHSCOPE_API_KEY | sk-你的API Key | 通义千问 API |
| DATABASE_URL | 你的数据库连接串 | Neon PostgreSQL |
| EMBEDDING_MODEL_NAME | text-embedding-v4 | Embedding 模型 |
| EMBEDDING_DIM | 1024 | 向量维度 |
| CHUNK_SIZE | 500 | 文本分块大小 |
| CHUNK_OVERLAP | 80 | 分块重叠 |
| LLM_MODEL_NAME | qwen3.5-plus | LLM 模型 |

**注意事项：**
- 环境变量会在创空间启动时自动注入
- 不要在代码中硬编码敏感信息
- 修改环境变量后需要重启创空间

### 4.4 获取创空间地址

创建成功后，你会得到一个类似这样的地址：

```
https://modelscope.cn/studios/你的用户名/创空间名称
```

## 常见问题

### Q1：部署后页面空白怎么办？

**原因：** 前端资源路径错误或未正确构建

**解决方案：**

```bash
# 1. 重新构建前端
npm run build

# 2. 检查 dist 目录
ls -la dist/

# 3. 确保 backend.Dockerfile 正确复制了 dist
# 在 Dockerfile 中添加：
COPY dist/ /home/user/app/dist/
```

### Q2：CORS 错误怎么办？

**原因：** 前端请求被后端 CORS 策略阻止

**解决方案：**

编辑 `api/main.py`，确保包含魔搭域名：

```python
allow_origins=[
    "https://modelscope.cn",
    "https://*.modelscope.cn",
    # ... 其他域名
]
```

### Q3：数据库连接失败怎么办？

**原因：** 数据库连接串错误或网络问题

**解决方案：**

1. 检查 `DATABASE_URL` 是否正确
2. 确保 Neon 数据库已启用 pgvector 扩展
3. 检查数据库是否在同一个区域（建议选择新加坡）

```sql
-- 在 Neon SQL Editor 中测试连接
SELECT 1;

-- 检查 pgvector 扩展
SELECT * FROM pg_extension WHERE extname = 'vector';
```

### Q4：API Key 无效怎么办？

**原因：** DashScope API Key 错误或未开通服务

**解决方案：**

1. 检查 API Key 是否正确复制（注意空格）
2. 确保已在 DashScope 控制台开通模型服务
3. 检查 API Key 是否有调用权限

```bash
# 测试 API Key
curl -X POST https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation \
  -H "Authorization: Bearer sk-你的API Key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen-plus",
    "input": {
      "messages": [{"role": "user", "content": "Hello"}]
    }
  }'
```

### Q5：内存不足怎么办？

**原因：** 资源配置过低

**解决方案：**

1. 在创空间设置中升级资源配置
2. 推荐使用 **4vCPU, 32GB 内存**
3. 或者优化代码，减少内存占用

### Q6：部署超时怎么办？

**原因：** 依赖安装或构建时间过长

**解决方案：**

1. 使用预构建的 Docker 镜像
2. 优化 `requirements.txt`，移除不必要的依赖
3. 使用国内镜像源：

```dockerfile
# 在 Dockerfile 中添加
RUN pip install --no-cache-dir -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple/
```

## 性能优化建议

### 1. 启用缓存

在 `api/main.py` 中添加缓存中间件：

- ✅ 添加速率限制
- ✅ 验证用户输入

### backend.Dockerfile 文件

```dockerfile
FROM modelscope-registry.cn-beijing.cr.aliyuncs.com/modelscope-repo/python:3.10

WORKDIR /home/user/app

# 安装 Node.js（用于构建前端）
RUN curl -fsSL https://deb.nodesource.com/setup_18.x | bash - && \
    apt-get install -y nodejs

# 复制前端依赖文件
COPY package.json package-lock.json ./

# 安装前端依赖
RUN npm install --registry=https://registry.npmmirror.com

# 复制前端源码
COPY src/ ./src/
COPY public/ ./public/
COPY index.html vite.config.ts tsconfig.json ./

# 构建前端
RUN npm run build

# 安装后端依赖
COPY api/requirements.txt ./api/
RUN pip install --no-cache-dir -r api/requirements.txt -i https://mirrors.aliyun.com/pypi/simple/

# 复制后端源码
COPY api/ ./api/

# 暴露端口
EXPOSE 7860

# 启动命令
CMD ["uvicorn", "api.main:app", "--host", "0.0.0.0", "--port", "7860"]
```
