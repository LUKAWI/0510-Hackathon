# 学科知识整合智能体 · 生产环境部署指南

> **预计时间：** 30-45 分钟（不包括等待构建）
> **费用：** 完全免费（前端 Vercel + 后端魔搭创空间 + 数据库 Neon，全部使用免费额度）

---

```
你的浏览器
    │
    ▼
┌─────────────────────────────────┐
│ ① 前端（Vercel 托管）              │  ← 本节部署
```

## 第一步：准备工作（5 分钟）

### 1.1 确认本地能正常运行

打开两个终端窗口：

**终端 A（后端）：**
```powershell
cd "d:\辣椒油\AI_project\Trae CN\hackathon\api"
uvicorn main:app --reload --port 8000
```
看到 `Uvicorn running on http://127.0.0.1:8000` 说明启动成功。

**终端 B（前端）：**
```powershell
cd "d:\辣椒油\AI_project\Trae CN\hackathon"
npm run dev
```
看到 `Local: http://localhost:5173/` 说明启动成功。

打开浏览器访问 `http://localhost:5173`，随便操作一下（上传教材、点知识图谱、问一个问题），确认没有报错。

### 1.2 确认数据库可用

1. 浏览器打开 [Neon Console](https://console.neon.tech/)，登录你的账号
2. 确认你的项目状态是 **Active**（绿色圆点），不是 Idle
3. 点击左侧 **SQL Editor**，输入以下 SQL 并执行：
   ```sql
   CREATE EXTENSION IF NOT EXISTS vector;
   ```
   如果返回 `CREATE EXTENSION` 或 `NOTICE: extension "vector" already exists`，说明 pgvector 已就绪

4. 点击左侧 **Dashboard**，找到 **Connection string**，复制完整连接串。格式类似：
   ```
   postgresql://username:password@ep-xxxx.us-east-2.aws.neon.tech/dbname?sslmode=require
   ```
   **把这个连接串保存好，后面要用！**

### 1.3 注册魔搭账号并获取 Token

> 魔搭是阿里云旗下的 AI 模型社区，提供免费的 Docker 创空间用于部署应用。

1. 浏览器打开 [魔搭社区](https://modelscope.cn)
2. 点击右上角 **登录/注册**，可以用：
   - 阿里云账号直接登录（推荐）
   - 手机号注册
3. 登录后，点击右上角头像 → **个人设置** → **访问令牌**
   - 或直接访问：https://modelscope.cn/my/myaccesstoken
4. 点击 **创建令牌**，给令牌起个名字（比如 `deploy`），复制生成的 Token
   - Token 是一长串字符，类似 `ms-xxxxxxxxxxxxxxxxxxxx`
5. **把这个 Token 保存好，后面要用！**

> ⚠️ **重要：** Docker 创空间要求实名认证。如果你用的是阿里云账号登录，一般已经实名了。如果没有，按照提示完成实名认证（需要身份证和支付宝）。

### 1.4 确认部署文件已就绪

这些文件我已经帮你创建好了，检查一下：

| 文件 | 位置 | 用途 |
|------|------|------|
| `vercel.json` | 项目根目录 | Vercel 前端部署配置（含框架声明防止误装 Python） |
| `.vercelignore` | 项目根目录 | 排除 `api/` 等后端文件，防止 Vercel 误装 Python 依赖 |

## 第三步：部署前端到 Vercel（10 分钟）

### 3.1 安装 Vercel CLI

打开终端，执行：

```powershell
npm install -g vercel
```

安装完成后，确认版本：
```powershell
npx vercel --version
```

### 3.2 登录 Vercel

```powershell
npx vercel login
```

终端会提示 "Press Enter to open the browser"，按回车。浏览器会自动打开 Vercel 登录页面，选择 **Continue with GitHub** 授权登录。

登录成功后终端显示 `Vercel CLI is now ready!`。

### 3.3 首次部署

在项目根目录执行：

```powershell
cd "d:\辣椒油\AI_project\Trae CN\hackathon"
npx vercel
```

CLI 会依次问你几个问题，按下面回答：

| 问题 | 回答 | 说明 |
|------|------|------|
| Set up and deploy? | **Y** (回车) | 确认部署 |
| Which scope? | 选择你的账号 | 用方向键选择，回车确认 |
| Link to existing project? | **N** (回车) | 首次部署，不关联已有项目 |
| What's your project's name? | `knowledge-agent` | 输入你喜欢的名字，回车 |
| In which directory is your code located? | **./** (直接回车) | 当前目录 |
| Override settings? | **N** (回车) | 自动识别为 Vite 项目 |

然后等待构建完成（约 1-2 分钟），终端会显示类似：

```
🔗  Preview URL: https://knowledge-agent-xxxxx.vercel.app
```

复制这个 URL，在浏览器打开，应该能看到你的前端页面。

> 这时候页面能显示，但 API 请求会失败（因为后端还没部署），这是正常的。

### 3.4 部署到生产环境

预览没问题后，执行生产部署：

```powershell
npx vercel --prod
```

会问 "Are you sure you want to deploy to production?"，输入 **Y** 回车。

完成后的 URL 就是你正式的前端地址，例如 `https://knowledge-agent.vercel.app`。**记下这个 URL！**

### 3.5 配置环境变量（先填占位值）

1. 浏览器打开 [Vercel Dashboard](https://vercel.com/dashboard)
2. 找到你刚创建的项目（`knowledge-agent`），点击进入
3. 点击顶部 **Settings** 标签 → 左侧 **Environment Variables**
4. 添加一个变量：

   | Key | Value | 勾选 Environments |
   |-----|-------|-------------------|
   | `VITE_API_URL` | `https://placeholder.example.com/api` | ✅ Production ✅ Preview ✅ Development |

5. 点击 **Save**
6. 页面会提示 "To apply to existing deployments, redeploy" → 点击 **Redeploy** 或在 Deployments 页找到最新部署点击右侧 `...` → **Redeploy**

> 注意：`VITE_API_URL` 现在先填假地址，等后端部署完成后，再回来改成真实地址并重新部署。

### 3.6 连接 GitHub 自动部署（可选但推荐）

这样以后你 push 代码，Vercel 自动重新部署，非常方便。

1. Vercel Dashboard → 你的项目 → **Settings** → **Git**
2. 点击 **Connect Git Repository**
3. 选择 **GitHub** → 授权 → 搜索并选择 `LUKAWI/0510-Hackathon`
4. 保持默认配置：
   - Framework Preset: **Vite**
   - Build Command: `npm run build`（覆盖默认）
   - Output Directory: `dist`（覆盖默认）
5. 点击 **Deploy**

完成！以后 `git push` 到 main 分支，Vercel 就会自动部署。

## 第五步：修复 CORS 配置并重新部署后端

### 5.1 更新 CORS 配置

当前 `api/main.py` 中的 CORS 只允许 `localhost:5173`，需要加上 Vercel 的前端域名。

打开 `d:\辣椒油\AI_project\Trae CN\hackathon\api\main.py`，找到第 48-57 行附近：

```python
# CORS 中间件（允许前端域名 + 本地开发）
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:5173",
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

在 `allow_origins` 列表里加上你的 Vercel 域名：

```python
# CORS 中间件（允许前端域名 + 本地开发）
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:5173",                        # 本地开发
        "https://knowledge-agent.vercel.app",           # Vercel 前端（替换成你的实际域名）
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 5.2 更新魔搭创空间中的代码

**如果用网页上传方式：**
1. 打开魔搭创空间 → **空间文件** 标签

## 常见问题排查

### Q1：前端页面打开了，但数据加载不出来？

**原因：** 前端连接不到后端。

**检查步骤：**
1. 确认魔搭创空间状态是"运行中"，不是"已停止"
2. 浏览器访问 `创空间URL/api/health`，确认返回 `{"status":"ok"}`
3. F12 → Network 标签，看 API 请求发到了哪个地址，确认和 `VITE_API_URL` 一致
4. 如果请求显示 CORS 错误（`blocked by CORS policy`），说明忘了第五步添加 Vercel 域名到 CORS 白名单

### Q2：魔搭创空间"运行错误"？

**检查方法：**
1. 进入创空间 → 点击 **"查看日志"** → 看 **运行日志**
2. 常见错误：
   - `could not translate host name` → 数据库连接串拼写错误
   - `DashScope embedding error` → API Key 错误或额度用完
   - `ModuleNotFoundError` → Dockerfile 中的 `pip install` 失败了

### Q3：上传文件后一直显示"处理中"？

**原因：** 后端处理教材需要调用 LLM API 提取知识点，可能需要几分钟。

**检查方法：**
1. 去魔搭创空间看运行日志，确认后台有在处理
2. 如果 5 分钟后还是处理中，可能是 API 调用失败，检查 `DASHSCOPE_API_KEY` 是否正确

### Q4：魔搭创空间说"构建失败"？

**常见原因和解决：**
- `pip install` 某个包失败 → 可能是网络波动，重新上线重试
- 镜像拉取超时 → 等几分钟再重新上线
- 磁盘空间不足 → 检查上传的文件不要太大（不要上传 `node_modules`！）

## 部署后维护

### 更新前端代码
```powershell
git add .
git commit -m "update: xxx"
git push
```
Vercel 自动检测到 push，自动构建和部署。

### 更新后端代码
在魔搭创空间 → 空间文件 → 替换修改后的文件 → 点击"上线"。
