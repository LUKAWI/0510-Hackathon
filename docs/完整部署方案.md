## 第一步：准备后端部署文件

### 1.1 确认已有文件

项目中已准备好以下部署文件，确认它们存在：

| 文件 | 路径 | 用途 |
|------|------|------|
| `backend.Dockerfile` | 项目根目录 | Docker 镜像构建配置 |
| `api/` | 项目根目录 | 后端 Python 代码（全部路由、服务、核心模块） |
| `api/requirements.txt` | api 目录 | Python 依赖列表 |

### 1.2 创建 ms_deploy.json

在项目根目录创建 `ms_deploy.json` 文件（如果不存在）：

```json
{
  "$schema": "https://modelscope.cn/api/v1/studios/deploy_schema.json",
  "sdk_type": "docker",
  "resource_configuration": "platform/2v-cpu-16g-mem",
  "port": 7860,
  "environment_variables": [
    { "name": "DASHSCOPE_API_KEY", "value": "sk-你的通义千问APIKey" },
    { "name": "DATABASE_URL", "value": "postgresql://user:pass@host/db?sslmode=require" },
    { "name": "EMBEDDING_MODEL_NAME", "value": "text-embedding-v4" },
    { "name": "EMBEDDING_DIM", "value": "1024" },
    { "name": "CHUNK_SIZE", "value": "500" },
    { "name": "CHUNK_OVERLAP", "value": "80" },
    { "name": "LLM_MODEL_NAME", "value": "qwen3.5-plus" }
  ]
}
```

**必须修改的两项：**

1. `DASHSCOPE_API_KEY`：替换为你的通义千问 API Key（从 [DashScope 控制台](https://dashscope.console.aliyun.com) 获取）
2. `DATABASE_URL`：替换为你的 Neon 数据库连接串（从 [Neon Console](https://console.neon.tech) 复制）

**其他参数说明（可保持默认）：**

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `EMBEDDING_MODEL_NAME` | `text-embedding-v4` | 通义千问 Embedding 模型 |
| `EMBEDDING_DIM` | `1024` | 向量维度（与模型匹配） |
| `CHUNK_SIZE` | `500` | 文本分块大小（字符数） |
| `CHUNK_OVERLAP` | `80` | 分块重叠长度 |
| `LLM_MODEL_NAME` | `qwen3.5-plus` | 大语言模型名称 |

### 1.3 确认 .gitignore 配置

确保 `ms_deploy.json` 已被加入 `.gitignore`，避免敏感信息泄露：

```gitignore
# 在 .gitignore 末尾添加
ms_deploy.json
.env
```

## 第二步：配置后端 CORS

### 2.1 修改 api/main.py

打开 `api/main.py`，找到 CORS 中间件配置（约第 52-61 行），将你的 Vercel 前端域名加入白名单：

```python
# CORS 中间件（允许前端域名 + 本地开发）
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:5173",                        # 本地开发
        "https://your-app.vercel.app",                  # Vercel 前端（替换成你的实际域名）
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

> ⚠️ **重要：** 将 `https://your-app.vercel.app` 替换为你的真实 Vercel 前端域名。如果忘记域名，可以在 Vercel Dashboard 的项目设置中查看。

## 后端 Dockerfile 说明

项目使用的 `backend.Dockerfile` 配置如下：

```dockerfile
FROM modelscope-registry.cn-beijing.cr.aliyuncs.com/modelscope-repo/python:3.10

WORKDIR /home/user/app

# 安装 Node.js（用于在构建时构建前端，当前方案不需要）
RUN curl -fsSL https://deb.nodesource.com/setup_18.x | bash - && \
    apt-get install -y nodejs && \
    apt-get clean && \
    rm -rf /var/lib/apt/lists/*

COPY package.json package-lock.json ./

RUN npm install --registry=https://registry.npmmirror.com

COPY src/ ./src/
COPY public/ ./public/
COPY index.html vite.config.ts tsconfig.json tsconfig.app.json tsconfig.node.json components.json ./

RUN npm run build

COPY api/requirements.txt ./api/requirements.txt

RUN pip install --no-cache-dir -r api/requirements.txt -i https://mirrors.aliyun.com/pypi/simple/

COPY api/ ./api/

COPY ms_deploy.json ./

EXPOSE 7860

ENV PYTHONPATH=/home/user/app
ENV HOST=0.0.0.0
ENV PORT=7860

CMD ["uvicorn", "api.main:app", "--host", "0.0.0.0", "--port", "7860"]
```

**构建流程说明：**

1. 基于 Python 3.10 镜像（魔搭国内源，下载快）
2. 安装 Node.js 18（用于前端构建）
3. 安装前端依赖并构建（生成 `dist/`）
4. 安装 Python 依赖（使用阿里云镜像加速）
5. 复制后端代码
6. 暴露 7860 端口（魔搭默认端口）
7. 启动 uvicorn 服务

> 💡 当前方案中，前端已经在 Vercel 部署，Dockerfile 中的前端构建步骤实际上不会生效（因为 `src/` 等前端文件没有上传到创空间）。如果只想部署后端，可以简化 Dockerfile，移除 Node.js 和前端构建部分。

## 常见问题排查

### Q1：前端页面打开了，但所有 API 请求都失败？

**排查步骤：**

1. 确认魔搭创空间状态是 **"运行中"**
2. 浏览器访问 `创空间URL/api/health`，确认返回 `{"status":"ok"}`
3. F12 → Network 标签，查看 API 请求的目标地址是否正确
4. 如果看到 `CORS error`，说明忘了在 `api/main.py` 中添加 Vercel 域名

**解决 CORS 问题：**

```python
# api/main.py 中
allow_origins=[
    "http://localhost:5173",
    "https://your-app.vercel.app",  # ← 添加这一行
],
```

修改后重新上传 `api/main.py` 到创空间并上线。

### Q2：上传教材后一直显示"处理中"？

**原因：** 后端调用通义千问 API 提取知识点需要时间。

**排查：**

1. 查看创空间运行日志，确认后台在处理
2. 如果 5 分钟后仍无进展，检查 `DASHSCOPE_API_KEY` 是否正确
3. 确认 DashScope 控制台有可用额度

### Q3：魔搭创空间构建失败？

**常见原因：**

| 错误 | 解决 |
|------|------|
| `pip install` 超时 | 重新上线重试（网络波动） |
| 磁盘空间不足 | 不要上传 `node_modules`、`src/` 等大文件 |
| 镜像拉取超时 | 等待几分钟后重试 |

### Q4：数据库连接失败？

**检查清单：**

1. Neon 数据库状态是否为 **Active**（不是 Idle）
2. 连接串是否包含 `?sslmode=require`
3. 是否已在 Neon SQL Editor 中执行 `CREATE EXTENSION IF NOT EXISTS vector;`
4. 创空间能否访问外网（魔搭默认可以）

**测试数据库连接：**

在 Neon SQL Editor 中执行：

```sql
-- 检查 pgvector 扩展
SELECT * FROM pg_extension WHERE extname = 'vector';

-- 检查表是否创建成功
SELECT table_name FROM information_schema.tables 
WHERE table_schema = 'public' 
ORDER BY table_name;
```

### Q5：RAG 问答返回空答案或报错？

**排查：**

1. 确认教材已成功索引（状态为"已索引"）
2. 查看创空间运行日志，确认 Embedding 调用成功
3. 检查向量表是否有数据：

```sql
-- 在 Neon SQL Editor 中执行
SELECT COUNT(*) FROM text_embeddings;  -- 表名可能不同，根据实际 schema 调整
```
